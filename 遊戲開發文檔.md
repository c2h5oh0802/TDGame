# 遊戲開發文檔：魔王守衛戰：流派覺醒

---

## 一、遊戲概述

### 1.1 遊戲簡介
**《魔王守衛戰：流派覺醒》**是一款結合「塔防 + 動作 + Roguelike」元素的網頁遊戲。玩家扮演魔王，在20波勇者軍團的攻勢下，利用防禦工事、召喚小兵、元素魔法等多種流派，保護核心與自身存活。

### 1.2 遊戲類型
- 塔防 (Tower Defense)
- 動作 (Action)
- Roguelike / Build 構建

### 1.3 技術架構
- **引擎**：原生 HTML5 Canvas 2D
- **語言**：JavaScript (ES6+)
- **UI 框架**：Tailwind CSS (CDN)
- **音效**：Web Audio API（8-bit 合成音效）
- **平台**：瀏覽器（桌面端）

### 1.4 遊戲目標
撐過 20 波敵人攻勢，最終擊敗「傳說勇者」Boss，或盡可能存活更久。

---

## 二、核心玩法

### 2.1 核心循環
```
移動/射擊 → 收集資源(金幣/經驗) → 升級抽卡 → 建造/強化 → 抵擋波次 → 重複
```

### 2.2 操作方式
| 按鍵 | 功能 |
|------|------|
| W/A/S/D | 移動魔王 |
| 滑鼠左鍵（按住） | 射擊 / 放置建築 |
| F | 靠近建築時升級/修復 |
| X | 靠近建築時拆除（退還金幣） |
| Z | 切換小兵模式（攻擊/防守/跟隨） |
| 空白鍵 | 提早開始下一波 |
| ESC | 暫停遊戲 |

### 2.3 遊戲狀態
- `START`：開始畫面
- `PLAYING`：正常遊戲
- `GAMEOVER`：遊戲結束（核心或玩家死亡）
- `LEVEL_UP`：升級抽卡
- `BUILDING`：建造模式
- `VICTORY`：勝利（擊敗 Boss）
- `PAUSED`：暫停

## 十五、遊戲平衡

### 15.1 難度曲線（心流理論 v1.2）

基於心流理論設計的四階段難度曲線，確保玩家始終處於「技能略低於挑戰」的甜蜜點：

| 階段 | 波次 | 難度乘數 | 生成數/次 | 掉落係數 | 設計目標 |
|------|------|----------|-----------|----------|----------|
| 學習期 | 1-5 | 0.3-1.5 | 1 | 100%+早期加成 | 熟悉操作、建立信心 |
| 成長期 | 6-10 | 2.0-4.0 | 1-2 | 100%+早期加成 | 穩定成長、建立經濟 |
| 挑戰期 | 11-15 | 5.0-9.0 | 2-5 | 75% | 技能測試、決策壓力 |
| 高潮期 | 16-20 | 10.8-18.0 | 5-9 | 50% | 極限挑戰、史詩感 |

#### 敵人類型出現時機
- 基礎敵人：第 1 波（100%）
- 遠程敵人：第 4 波（12%）
- 刺客敵人：第 6 波（15%）
- 術士敵人：第 7 波（18%）
- 攻城敵人：第 8 波（22%）
- 菁英敵人：第 10 波（25%）

### 15.2 敵人強度公式
```
// diffMult 心流曲線計算
波次 1-5:  diffMult = 波次 × 0.3     // 0.3, 0.6, 0.9, 1.2, 1.5
波次 6-10: diffMult = 1.5 + (波次-5) × 0.5  // 2.0, 2.5, 3.0, 3.5, 4.0
波次 11-15: diffMult = 4.0 + (波次-10) × 1.0 // 5.0, 6.0, 7.0, 8.0, 9.0
波次 16-20: diffMult = 9.0 + (波次-15) × 1.8 // 10.8, 12.6, 14.4, 16.2, 18.0

HP = 基礎值 + diffMult × 係數 + 玩家等級 × 係數
速度 = 基礎值 + 隨機值 + diffMult
傷害 = 基礎值 + diffMult × 係數
```

### 15.3 掉落公式
```
// dropNerf 心流曲線計算
波次 1-11:  dropNerf = 1.0   // 完整掉落
波次 12-15: dropNerf = 0.75  // 輕微懲罰
波次 16-20: dropNerf = 0.5   // 重度懲罰

// 早期金幣加成
波次 1-10: earlyBonus = 1.0 + (10-波次) × 0.15
波次 11+:  earlyBonus = 1.0

金幣 = 基礎值 × dropNerf × earlyBonus
經驗 = 基礎值 + diffMult × 0.5
```

### v1.2 (2026/04/05) - 心流難度曲線修正

#### 難度平衡修正

##### 1. 心流理論難度乘數 (diffMult)
- **舊公式**：`diffMult = 波次 × 0.8` (線性成長，第 8 波 = 6.4)
- **新公式**：分段式心流曲線
  - 學習期 (1-5)：`波次 × 0.3` → 0.3, 0.6, 0.9, 1.2, 1.5
  - 成長期 (6-10)：`1.5 + (波次-5) × 0.5` → 2.0, 2.5, 3.0, 3.5, 4.0
  - 挑戰期 (11-15)：`4.0 + (波次-10) × 1.0` → 5.0, 6.0, 7.0, 8.0, 9.0
  - 高潮期 (16-20)：`9.0 + (波次-15) × 1.8` → 10.8, 12.6, 14.4, 16.2, 18.0
- **效果**：第 8 波難度從 6.4 降為 3.0，大幅降低早期壓力

##### 2. 掉落懲罰延後 (dropNerf)
- **舊規則**：第 8 波開始 75% → 第 12 波 50%
- **新規則**：第 12 波開始 75% → 第 16 波 50%
- **效果**：玩家有更多時間建立經濟基礎

##### 3. 敵人生成數量心流化 (spawnCount)
- **舊公式**：`1 + 波次×0.3 + 波次²×0.015` (第 8 波 = ~4 隻)
- **新公式**：分段式
  - 學習期 (1-5)：每次 1 隻
  - 成長期 (6-10)：`1 + (波次-5)×0.4` → 1-2 隻
  - 挑戰期 (11-15)：`2 + (波次-10)×0.6` → 2-5 隻
  - 高潮期 (16-20)：`5 + (波次-15)×0.8` → 5-9 隻
- **效果**：早期壓力大幅降低，後期才有真正的群體壓力

##### 4. 敵人類型出現時機調整
- 遠程：第 3 波 15% → 第 4 波 12%
- 刺客：第 4 波 25% → 第 6 波 15%
